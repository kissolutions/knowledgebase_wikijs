# Lighting Hardware – Fundamentals

## Purpose of This Section
This section documents **lighting hardware as physical devices**, focusing on:
- What exists
- What constraints it imposes
- What capabilities it provides

It is intended to support:
- Design exploration
- System reasoning
- Architecture evaluation

This section does **not** recommend specific system architectures or design solutions.

---

## How to Use Hardware Pages

Hardware pages answer the question:

> **“What is this thing, how does it behave, and what constraints does it impose?”**

They are meant to help you:
- Determine whether a device category is relevant
- Understand the implications of choosing it
- Decide whether to continue up or down the abstraction chain

If you are asking:
- *“Should I use this here?”* → see **Lighting Design**
- *“How should this behave?”* → see **Lighting Controls**
- *“Which platform?”* → see **Architecture pages**
